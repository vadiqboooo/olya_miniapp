# Техническое задание: Админ-панель для создания программ тренировок

## 1. Общая информация

### 1.1 Цель проекта
Разработка административной панели для создания и управления программами тренировок в существующем фитнес-приложении.

### 1.2 Целевая аудитория
Администраторы и тренеры, которые будут создавать программы тренировок для пользователей приложения.

### 1.3 Технологический стек
- **Frontend**: React 19.2.0, React Router DOM v7.13.0, Vite, Axios
- **Backend**: FastAPI, SQLAlchemy, SQLite
- **Стиль**: Кастомный CSS (в соответствии с существующим дизайном)

---

## 2. Функциональные требования

### 2.1 Основной функционал

Админ-панель должна обеспечивать следующий функционал:

1. **Просмотр всех программ** - список всех созданных программ тренировок с возможностью фильтрации и поиска
2. **Создание программы** - трёхэтапный процесс создания программы тренировок:
   - Создание программы (базовые параметры)
   - Добавление дней тренировок к программе
   - Добавление упражнений к каждому дню
3. **Редактирование программ** - изменение параметров существующих программ, дней и упражнений
4. **Удаление программ** - удаление программ с подтверждением

---

## 3. Детальное описание функционала

### 3.1 Просмотр всех программ тренировок

#### 3.1.1 Страница списка программ
**Путь**: `/admin/programs`

**Элементы интерфейса**:

1. **Хедер страницы**:
   - Заголовок "Программы тренировок"
   - Кнопка "Создать программу" (primary action) → переход к `/admin/programs/create`
   - Поле поиска по названию программы

2. **Фильтры**:
   - Фильтр по уровню сложности (Все, Начальный, Средний, Продвинутый)
   - Фильтр по цели (Все, Похудение, Набор массы, Выносливость, Гибкость)
   - Фильтр по месту (Все, Дома, Зал, Улица)
   - Кнопка "Сбросить фильтры"

3. **Таблица/Grid программ**:
   Каждая карточка программы содержит:
   - **Название программы** (крупный текст)
   - **Теги**: уровень, цель, место (в виде badges)
   - **Описание** (первые 100 символов с многоточием)
   - **Статистика**:
     - Количество дней тренировок (например, "7 дней")
     - Общее количество упражнений (например, "25 упражнений")
   - **Действия** (кнопки/иконки):
     - "Редактировать" → `/admin/programs/{program_id}/edit`
     - "Управление днями" → `/admin/programs/{program_id}/days`
     - "Удалить" (с модальным подтверждением)
     - "Предпросмотр" (как видят пользователи) - опционально

4. **Пагинация** (если программ больше 20):
   - Пагинация внизу страницы
   - Показывать по 20 программ на странице

**Состояния**:
- **Loading**: Отображение Loader при загрузке
- **Empty state**: "Программы не найдены. Создайте первую программу!" + кнопка "Создать программу"
- **No results**: "По заданным фильтрам ничего не найдено" (при активных фильтрах)

**Поиск и фильтрация**:
- Поиск работает в реальном времени (debounce 300ms)
- Фильтры можно комбинировать
- URL обновляется при изменении фильтров (для возможности делиться ссылками)
  Пример: `/admin/programs?difficulty=Начальный&goal=Похудение&search=кардио`

**API запросы**:
- `GET /programs/` - получение всех программ с query параметрами:
  - `?difficulty=string` - фильтр по уровню
  - `?goal=string` - фильтр по цели
  - `?location=string` - фильтр по месту
  - `?search=string` - поиск по названию
  - `?page=int&limit=int` - пагинация (опционально)
- `DELETE /programs/{program_id}` - удаление программы

**Модальное окно подтверждения удаления**:
- Заголовок: "Удалить программу?"
- Текст: "Вы уверены, что хотите удалить программу '{название программы}'? Это действие нельзя отменить. Будут удалены все дни и упражнения."
- Кнопки:
  - "Отмена" (secondary)
  - "Удалить" (danger, красная)

### 3.2 Редактирование программы

#### 3.2.1 Форма редактирования программы
**Путь**: `/admin/programs/{program_id}/edit`

**Элементы интерфейса**:

1. **Хедер страницы**:
   - Заголовок: "Редактирование программы"
   - Навигация: "Программы" → "Редактирование"
   - Кнопка "Отмена" (возврат к списку программ)

2. **Форма редактирования** (идентична форме создания):
   | Поле | Тип | Обязательность | Описание |
   |------|-----|----------------|----------|
   | Название программы | Text input | Обязательное | Название программы |
   | Описание программы | Textarea | Опциональное | Подробное описание программы |
   | Уровень сложности | Select | Обязательное | Варианты: "Начальный", "Средний", "Продвинутый" |
   | Цель тренировок | Select | Обязательное | Варианты: "Похудение", "Набор массы", "Выносливость", "Гибкость" |
   | Место занятий | Select | Обязательное | Варианты: "Дома", "Зал", "Улица" |

3. **Дополнительная информация** (read-only):
   - Дата создания программы
   - Количество дней в программе
   - Количество упражнений

**Валидация**:
- Такая же, как при создании программы
- Название программы: минимум 3 символа, максимум 100 символов
- Описание: максимум 1000 символов
- Все select-поля должны иметь выбранное значение

**Поведение**:
- При загрузке страницы форма заполняется текущими данными программы
- Отображается индикатор загрузки до получения данных
- Кнопка "Сохранить" активна только при изменении данных

**Действие после отправки**:
- PUT запрос к API `/programs/{program_id}` с обновленными данными
- При успешном обновлении:
  - Показать уведомление "Программа успешно обновлена"
  - Варианты действия (на выбор пользователя):
    - "Вернуться к списку программ"
    - "Перейти к управлению днями"
    - Остаться на странице редактирования

**UI элементы**:
- Кнопка "Сохранить изменения" (primary action)
- Кнопка "Отмена" (возврат к списку без сохранения)
- Кнопка "Перейти к дням" (быстрый переход к управлению днями)

**API запросы**:
- `GET /programs/{program_id}` - получение данных программы
- `PUT /programs/{program_id}` - обновление программы

**Связанные действия**:
- После редактирования базовой информации можно сразу перейти к редактированию дней тренировок
- Ссылка "Управление днями" в хедере для быстрого перехода

| Уровень сложности | Select | Обязательное | Варианты: "Начальный", "Средний", "Продвинутый" |

- Название программы: минимум 3 символа, максимум 100 символов

### 3.4 Добавление и редактирование дней тренировок

#### 3.4.1 Страница управления днями
**Путь**: `/admin/programs/{program_id}/days`

**Элементы интерфейса**:

1. **Хедер страницы**:
   - Название программы
   - Кнопка "Завершить создание программы" (переход к списку всех программ)
   - Краткая информация о программе (уровень, цель, место)

2. **Список дней** (отображение существующих дней):
   - Карточки дней в виде списка или grid
   - Каждая карточка содержит:
     - Номер дня (автоматически: День 1, День 2, и т.д.)
     - Название тренировки
     - Количество упражнений (например, "5 упражнений")
     - Кнопка "Редактировать упражнения" → переход к `/admin/programs/{program_id}/days/{workout_id}/exercises`
     - Кнопка "Удалить день" (с подтверждением)

3. **Форма добавления нового дня**:
   | Поле | Тип | Обязательность | Описание |
   |------|-----|----------------|----------|
   | Название тренировки | Text input | Обязательное | Например, "Кардио тренировка", "День отдыха" |
   | Описание дня | Textarea | Опциональное | Краткое описание дня тренировок |

4. **Кнопки действий**:
   - "Добавить день" (добавляет день и остаётся на текущей странице)
   - "Добавить день и перейти к упражнениям" (добавляет день и сразу переходит к добавлению упражнений)

**Поведение**:
- Дни автоматически нумеруются (`day_number`) в порядке добавления
- После добавления дня форма очищается
- Список дней обновляется динамически
- Можно добавить неограниченное количество дней

**API запросы**:
- `GET /workouts/{program_id}` - загрузка существующих дней
- `POST /workouts/` - создание нового дня
- `PUT /workouts/{workout_id}` - обновление дня
- `DELETE /workouts/{workout_id}` - удаление дня

#### 3.4.2 Редактирование дня тренировки

**Способ редактирования**: Inline редактирование или модальное окно

**Вариант 1: Inline редактирование** (рекомендуется):
- При клике на карточку дня она переходит в режим редактирования
- Поля название и описание становятся редактируемыми
- Появляются кнопки "Сохранить" и "Отмена"
- После сохранения карточка возвращается в обычный режим

**Вариант 2: Модальное окно**:
- При клике на кнопку "Редактировать" открывается модальное окно
- Форма с полями:
  - Название тренировки
  - Описание дня
- Кнопки "Сохранить" и "Отмена"

**Поля для редактирования**:
| Поле | Тип | Обязательность |
|------|-----|----------------|
| Название тренировки | Text input | Обязательное |
| Описание дня | Textarea | Опциональное |

**Примечание**: `day_number` не редактируется (автоматически определяется порядком)

**Поведение**:
- При успешном обновлении показывается уведомление
- Список дней обновляется автоматически
- Валидация такая же, как при создании

**API запрос**:
- `PUT /workouts/{workout_id}` - обновление дня

### 3.5 Добавление и редактирование упражнений

#### 3.5.1 Страница управления упражнениями
**Путь**: `/admin/programs/{program_id}/days/{workout_id}/exercises`

**Элементы интерфейса**:

1. **Хедер страницы**:
   - Навигация: Название программы → День X → Упражнения
   - Кнопка "Вернуться к дням"
   - Название текущего дня тренировок

2. **Список упражнений** (отображение существующих упражнений):
   - Карточки упражнений в виде списка
   - Каждая карточка содержит:
     - Название упражнения
     - Подходы × Повторения (например, "3 × 10-15")
     - Время выполнения (если указано)
     - Время отдыха (например, "60 сек")
     - Описание (если указано)
     - Кнопка "Редактировать" (inline редактирование)
     - Кнопка "Удалить" (с подтверждением)
   - Drag & drop для изменения порядка упражнений (опционально)

3. **Форма добавления упражнения**:
   | Поле | Тип | Обязательность | Описание |
   |------|-----|----------------|----------|
   | Название упражнения | Text input | Обязательное | Например, "Отжимания", "Приседания" |
   | Количество подходов | Number input | Обязательное | Целое число (например, 3) |
   | Количество повторений | Text input | Обязательное | Может быть диапазон или точное число (например, "10-15", "30 сек") |
   | Время выполнения | Number input | Опциональное | В секундах (например, 30 для планки) |
   | Время отдыха | Number input | Обязательное | В секундах между подходами (например, 60) |
   | Описание | Textarea | Опциональное | Техника выполнения, особенности |

4. **Кнопки действий**:
   - "Добавить упражнение" (добавляет и очищает форму)
   - "Сохранить и вернуться к дням" (возврат к списку дней)

**Валидация**:
- Название: минимум 2 символа, максимум 100 символов
- Подходы: от 1 до 10
- Повторения: обязательное поле, текстовый формат
- Время выполнения: 0-3600 секунд
- Время отдыха: 0-600 секунд
- Описание: максимум 500 символов

**API запросы**:
- `GET /workouts/{workout_id}/exercises` - загрузка упражнений
- `POST /exercises/` - создание нового упражнения
- `PUT /exercises/{exercise_id}` - обновление упражнения
- `DELETE /exercises/{exercise_id}` - удаление упражнения

#### 3.5.2 Редактирование упражнения

**Способ редактирования**: Inline редактирование в карточке упражнения

**Поведение**:
- При клике на кнопку "Редактировать" карточка упражнения переходит в режим редактирования
- Все поля становятся редактируемыми
- Появляются кнопки "Сохранить" и "Отмена"
- При сохранении данные отправляются на сервер
- После успешного сохранения карточка возвращается в режим просмотра

**Форма редактирования** (те же поля, что и при создании):
| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| Название упражнения | Text input | Обязательное | Название упражнения |
| Количество подходов | Number input | Обязательное | Целое число (1-10) |
| Количество повторений | Text input | Обязательное | Диапазон или точное число |
| Время выполнения | Number input | Опциональное | В секундах (0-3600) |
| Время отдыха | Number input | Обязательное | В секундах (0-600) |
| Описание | Textarea | Опциональное | Техника выполнения |

**Валидация**:
- Такая же, как при создании упражнения
- Все изменения валидируются перед отправкой

**API запрос**:
- `PUT /exercises/{exercise_id}` - обновление упражнения

**UI элементы**:
- Кнопка "Сохранить" (primary, появляется в режиме редактирования)
- Кнопка "Отмена" (отменяет изменения и возвращает исходные данные)


## 5. API Endpoints

### 5.1 Существующие endpoints (можно использовать)

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/programs/` | Получить все программы (поддерживает query параметры: difficulty, goal, location) |
| GET | `/programs/{program_id}` | Получить конкретную программу |
| POST | `/programs/` | Создать программу |
| GET | `/workouts/{program_id}` | Получить все дни программы |
| GET | `/workouts/{workout_id}/exercises` | Получить упражнения дня |

**Примечание**: Endpoint `GET /programs/` нужно расширить для поддержки:
- `?search=string` - поиск по названию программы (LIKE или ILIKE)
- `?page=int&limit=int` - пагинация (опционально)
- Должен возвращать также счетчик дней и упражнений для каждой программы

```python
# Обновление программы

Request Body:
```

```python
# Создание упражнения
POST /exercises/
Request Body:
{

  "sets": int,
  "reps": "string",
```

   - Цель: Похудение
   - Место: Дома
5. Нажать "Создать программу"
6. Добавить 2 дня тренировок:
   - День 1: "Кардио тренировка"
   - День 2: "Силовая тренировка"
7. К первому дню добавить 3 упражнения с разными параметрами
8. Ко второму дню добавить 2 упражнения
9. Вернуться к списку программ
10. Проверить, что "Тест программа" отображается в списке
11. Проверить статистику: "2 дня", "5 упражнений"

3. Добавить фильтр "Цель: Похудение"

5. Использовать поиск, ввести "Тест"
6. Проверить, что найдена "Тест программа"
7. Нажать "Сбросить фильтры"
8. Проверить, что отображаются все программы

**Сценарий 3: Редактирование программы**
1. Найти "Тест программа" в списке

6. Проверить, что отображается уведомление об успехе

8. Проверить, что название и теги обновились

**Сценарий 4: Редактирование дня и упражнений**
1. Открыть "Обновленная тест программа"
2. Перейти к управлению днями

4. Сохранить изменения

6. Отредактировать одно из упражнений, изменить количество подходов
7. Сохранить изменения
8. Проверить, что изменения отображаются корректно

**Сценарий 5: Удаление элементов**
1. В упражнениях удалить одно упражнение

6. Подтвердить удаление
